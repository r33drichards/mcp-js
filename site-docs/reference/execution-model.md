# Execution Model Reference

## Execution Statuses

| Status | String Value | Description |
|--------|-------------|-------------|
| Running | `running` | Code is currently executing in a V8 isolate |
| Completed | `completed` | Execution finished successfully |
| Failed | `failed` | Execution encountered an error |
| Cancelled | `cancelled` | Execution was cancelled by user via `cancel_execution` |
| Timed Out | `timed_out` | Execution exceeded the timeout limit |

## Execution Lifecycle

```
┌──────────┐     ┌───────────┐     ┌───────────┐
│ Submitted │────>│  Running   │────>│ Completed │
└──────────┘     └───────────┘     └───────────┘
                       │
                       ├────────────>┌───────────┐
                       │             │  Failed    │
                       │             └───────────┘
                       │
                       ├────────────>┌───────────┐
                       │             │ Cancelled  │
                       │             └───────────┘
                       │
                       └────────────>┌───────────┐
                                     │ Timed Out  │
                                     └───────────┘
```

1. **Submitted**: `run_js` is called. TypeScript types are stripped via SWC. A UUID execution ID is generated. The execution is registered with status `running`.
2. **Running**: A V8 semaphore permit is acquired (bounds concurrency to `--max-concurrent-executions`). Code executes on the Tokio blocking thread pool. The V8 isolate handle is published for cancellation.
3. **Terminal state**: One of `completed`, `failed`, `cancelled`, or `timed_out`.

## Execution ID Format

Execution IDs are UUIDs generated by `uuid::Uuid::new_v4()`.

Example: `a1b2c3d4-e5f6-4789-abcd-ef0123456789`

## Cancellation

Cancellation uses V8's `IsolateHandle::terminate_execution()` method.

- The isolate handle is stored in the execution registry after V8 starts
- `cancel_execution(id)` calls `handle.terminate_execution()` on the V8 isolate
- The V8 isolate throws an uncatchable termination exception
- Status transitions to `cancelled` with error message `"Cancelled by user"`
- Cancellation of non-running executions returns an error

## Timeout Enforcement

Timeouts use `tokio::select!` racing the V8 blocking task against a `tokio::time::sleep`:

```
tokio::select! {
    res = &mut join_handle => { /* execution completed */ }
    _ = tokio::time::sleep(timeout_dur) => {
        handle.terminate_execution();  // terminate V8
        join_handle.await;             // wait for cleanup
    }
}
```

- Default timeout: 30 seconds (`DEFAULT_EXECUTION_TIMEOUT_SECS`)
- Maximum timeout: 300 seconds (enforced by clap parser)
- Minimum timeout: 1 second
- Per-execution override via `execution_timeout_secs` parameter
- Status transitions to `timed_out` with error message `"Execution timed out"`

## Concurrency Control

- Maximum concurrent V8 executions is controlled by `--max-concurrent-executions` (default: CPU core count)
- Enforced via a `tokio::sync::Semaphore`
- Stateful (snapshot) executions are additionally serialized by a `tokio::sync::Mutex` because V8's SnapshotCreator is not thread-safe

## Out of Memory Handling

When V8 approaches the heap limit:

1. `near_heap_limit_callback` fires, sets the OOM flag, calls `isolate.terminate_execution()`
2. The heap limit is doubled (to allow V8 to unwind gracefully)
3. The termination error is classified as OOM based on the flag
4. Error message: `"Out of memory: V8 heap limit exceeded. Try increasing heap_memory_max_mb."`

If V8 hits an unrecoverable OOM (e.g., structural allocation limits exceeded), the `oom_error_handler` logs and calls `std::process::abort()`.

## ExecutionInfo Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Execution UUID |
| `status` | `string` | One of: `running`, `completed`, `failed`, `cancelled`, `timed_out` |
| `result` | `string?` | Return value (present when `completed`) |
| `heap` | `string?` | Output heap snapshot hash (stateful mode only) |
| `error` | `string?` | Error message (present when `failed`, `cancelled`, or `timed_out`) |
| `started_at` | `string` | ISO-8601 timestamp |
| `completed_at` | `string?` | ISO-8601 timestamp (absent while `running`) |

## ExecutionSummary Fields

| Field | Type | Description |
|-------|------|-------------|
| `execution_id` | `string` | Execution UUID |
| `status` | `string` | Current status |
| `started_at` | `string` | ISO-8601 timestamp |
| `completed_at` | `string?` | ISO-8601 timestamp |
